# The-real-bag — Shopify Theme

Shopify store for **HerVoyage**, a travel/garment-bag brand. Built on top of a stock Shopify theme template (likely Dawn-derived), so the repo contains many files inherited from the base theme that **are not used by this store**.

## What's actually in use

Only two pages are live, and only a small subset of files power them. When making changes, work within these files unless explicitly told otherwise.

### Homepage
- Template: [templates/index.json](templates/index.json)
- Sections: anything in [sections/](sections/) prefixed with `sitg-` (e.g. `sitg-hero.liquid`, `sitg-marquee.liquid`, `sitg-reframe.liquid`, `sitg-ingredients.liquid`, `sitg-science.liquid`, `sitg-comparison.liquid`, `sitg-timeline.liquid`, `sitg-trust.liquid`, `sitg-founder.liquid`, `sitg-testimonials.liquid`, `sitg-purchase.liquid`, `sitg-faq.liquid`, `sitg-final-cta.liquid`)
- Header/footer: [sections/sitg-header.liquid](sections/sitg-header.liquid), [sections/sitg-footer.liquid](sections/sitg-footer.liquid)
- Shared landing styles: `assets/sitg-landing.css`

### Product page (Luhxe layout)
- Template: [templates/product.luhxe.json](templates/product.luhxe.json)
- Section: [sections/main-product-luhxe.liquid](sections/main-product-luhxe.liquid)
- Styles: [assets/main-product-luhxe.css](assets/main-product-luhxe.css)
- The product template also reuses several `sitg-*` sections below the main product block (marquee, reframe, ingredients, science, comparison, timeline, trust, founder, testimonials, faq, final-cta).

## What to ignore

Everything else in [sections/](sections/), [templates/](templates/), [assets/](assets/), [snippets/](snippets/), etc. is leftover Dawn/base-theme scaffolding. Treat it as dead code unless the user explicitly references it. Don't refactor or "clean up" unused files — Shopify's theme editor and presets may still reference them in ways that aren't obvious from a code search.

## Before editing: check for unpulled remote changes

The merchant frequently edits this theme directly in the Shopify admin theme editor, which pushes commits to the remote (`main`) without those changes existing locally. If we edit locally on top of a stale `main`, the next push will conflict.

**Before making any code change in this repo**, run `git fetch` and check whether the local branch is behind the remote (e.g. `git status -sb` showing `behind` count, or comparing `HEAD` to `origin/main`). If the local branch is behind:

1. Stop before editing.
2. Tell the user how many commits behind we are and (briefly) what changed on the remote.
3. Ask the user whether to pull the latest changes before continuing with their request.
4. Only proceed with the edits after the user confirms — either pulling first, or explicitly choosing to edit anyway.

If the local branch is up to date, proceed normally without mentioning the check.

## Conventions

- Section JSON in `templates/*.json` is **auto-generated by the Shopify admin theme editor**. Edits made locally can be overwritten when the merchant edits the theme in Shopify admin. When changing copy or settings, prefer editing schema defaults in the `.liquid` section file so future re-syncs preserve intent — but ask first if unsure which side is the source of truth for a given change.
- Custom sections use the `sitg-` prefix. Keep that convention for new sections that belong to this brand's landing pages.
- The product page uses the `luhxe` suffix (`main-product-luhxe`, `product.luhxe.json`) to distinguish it from the inherited `main-product.liquid`.
